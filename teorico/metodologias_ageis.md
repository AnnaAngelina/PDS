# Metodologias ágeis

Mais chance de sucesso que o método tradicional

- Processos dirigidos a planos:

Planejamento de atividades a fim de desenvolver 

- Processos ágeis:

Incremento de tarefas, ajustes para refletir as necessidades dos clientes

- Processo de software

conjunto de atividades que estabelecem metodologias para a prática da engenharia de software

## O processo tradicional (cascata)

Planejamento de tarefas sucessivas e bem definidas formando um todo não flexível e, por isso, são elaboradas com rigor. 
Nele, há pouca interação com o cliente implicando que muitas das funcionalidades não agregarão valor ao cliente, enquanto as necessárias muitas vezes não foram implementadas

``
levantar requisitos --> modelagem --> implementação --> teste e validação --> entrega
``

Esse modelo não se adapta a volatidade das regras de negócio, assim como a entregas rápidas que não inviáveis por sistemas baseados em especificação
detalhada dos requisitos, projeto e testes.

## Processos ágeis

Necessita de softwares prontos rapidadente.

- Manifesto ágil: movimento inicialdo por programadores (80-90) que possuia como princípios:
a) indivíduos e interação
b) software funcionando
c) colaboração com o cliente
d) adaptação a mudanças

- Princípios: satisfação, mudanças tardias, software funcionando, comunicação física é mais eficiente

- Desvantagens: alta criticidade, necessita de desenvolvedores experientes, documentação mínima, mudança de equipe

### O manifesto ágil

Depois surgiram várias metodologias --> XP, SCRUM, LEAN

#### XP

Um dos mais conhecidos métodos ágeis que ajuda a criar sistemas de melhor qualidade, que são
produzidos em menos tempo e de forma mais econômica que o habitual. Ele é indicado para equipes médias e pequenas;

- Atividades metodológicas:
  Planejamento
  Projeto
  Codificação
  Testes

#### SCRUM

Planejamento de projetos de software em uma equipe que colabora, trabalha junto. Possui um quaro de funções e tarefas que serão acompanhadas diariamente.

- Três pilares:
  Transparência
  Inspeção
  Adaptação
