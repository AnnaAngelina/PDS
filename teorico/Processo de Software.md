# Processo de Software

É uma abordagem adaptável que possibilita às pessoas (a
equipe de software) realizar o trabalho de selecionar e
escolher o conjunto apropriado de ações e tarefas;

A essência da prática:

• Compreender o problema (comunicação e análise);

• Planejar uma solução (modelagem e projeto de software);

• Executar o plano (geração do código);

• Examinar o resultado para ter precisão (testes e garantia da
qualidade).

## Princípios gerais - 

1. A razão de existir:
   
  Um sistema de software existe por uma razão: gerar valor a seus usuários

2. KISS (Keep It Simple, Stupid! – Faça de forma simples, estúpido!):
   
  Todo o projeto deve ser o mais simples possível, mas não tão simples assim.

3. Mantenha a visão:
   
  Uma visão clara é essencial para o sucesso. Mantenha a integridade
  conceitual, não faça uma colcha de retalhos.

4. O que um produz, os outros consomem:
   
  Sempre especifique, projete e implemente ciente de que alguém mais
  tarde terá que entender o que você está fazendo.
